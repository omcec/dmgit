# Fiscal

### Sintegra <a href="#sintegra" id="sintegra"></a>

{% tabs %}
{% tab title="Sintegra" %}
Sintegra é um arquivo gerado para informar os registros de entrada e saída de uma empresa. Após a geração do arquivo é necessário importar o arquivo para o validador para visualizar e comparar as informações com os relatórios gerados pelo sistema. Caso tenha alguma informação inconsistente é necessário corrigir.

Este módulo é utilizado para gerar o arquivo Sintegra do estabelecimento.

#### Gerar arquivo <a href="#gerar-arquivo" id="gerar-arquivo"></a>

Ao acessar o módulo, é exibida a tela com os dados cadastrais do estabelecimento e os parâmetros para geração do arquivo, sendo necessário apenas inserir o **nome de contato** e **telefone**. Após inserir as informações, clicar no botão **Gerar Arquivo**. **Observações:**

Ao abrir o aplicativo, o mês e ano de referência corretos já são selecionados automaticamente. Isso acontece porque o aplicativo carrega o último mês já concluído.

As especificações já são marcadas por padrão, mas podem ser alteradas de acordo com a necessidade do cliente.

#### Validar Arquivo <a href="#validar-arquivo" id="validar-arquivo"></a>

### SPED ICMS <a href="#sped-icms" id="sped-icms"></a>

{% tabs %}
{% tab title="SPED ICMS" %}
O módulo é utilizado para gerar o arquivo SPED do estabelecimento.

**Observação 1:** o módulo só é liberado para acesso para clientes que possuem o serviço de geração de SPED contratado com a Dmaster, porém o serviço não tem custo. Será necessário apenas encaminhar o chamado para o setor comercial para que inclua esse serviço no cadastro do cliente.

**Observação 2:** para realizar a geração do arquivo, os dados do contador devem ser inseridos no módulo dados do contador no DMConfigurações.

### Nota Fiscal Eletrônica <a href="#nota-fiscal-eletronica" id="nota-fiscal-eletronica"></a>

{% tabs %}
{% tab title="Nota Fiscal Eletrônica" %}
Conceito

Podemos conceituar a Nota Fiscal Eletrônica como sendo um documento de existência apenas digital, emitido e armazenado eletronicamente, com o intuito de documentar, para fins fiscais, uma operação de circulação de mercadorias ou uma prestação de serviços, ocorrida entre as partes. Sua validade jurídica é garantida pela assinatura digital do remetente (garantia de autoria e de integridade) e a Autorização de uso fornecida pelo Fisco, antes da ocorrência do fato gerador.

#### DANFE <a href="#danfe" id="danfe"></a>

O DANFE (Documento Auxiliar da Nota Fiscal Eletrônica) é uma representação gráfica da NF-e, mas não substitui e não se confunde com a nota fiscal eletrônica, sendo apenas uma de suas várias representações possíveis na forma impressa.

#### Situações da NF-e <a href="#situacoes-da-nf-e" id="situacoes-da-nf-e"></a>

Durante a emissão de uma Nota Fiscal eletrônica, esta passa por diversas situações até ser autorizada pela Secretaria de Fazenda Estadual (SEFAZ). Estas situações serão citadas e explicadas abaixo:

* **Em digitação:** o registro acaba de ser criado e ainda não está completamente finalizado com os dados. Pode ser editado a qualquer hora;
* **Validada:** apenas após a inserção de todas as informações necessárias da NF-e e a realização da operação de Validação a NF-e ficará com a situação **Validada**. Nesta situação, a NF-e encontra-se corretamente preenchida com relação à estrutura de dados. Caso a NF-e seja editada, deverá passar novamente pelo processo de validação;
* **Assinada:** é a situação onde a NF-e está após ser assinada digitalmente com o Certificado Digital do emissor. Ainda é possível a edição desta NF-e, mas esta voltará para a situação **Em Digitação**, perdendo a assinatura e sendo necessário realizar novamente os processos de Validação e Assinatura;
* **Em processamento na SEFAZ:** após a transmissão da NF-e para a SEFAZ, a mesma ficará com a situação **Em processamento na SEFAZ**. Neste estágio, a NF-e estará na fila para ser processada na SEFAZ, sendo necessário aguardar até que a SEFAZ recepcione e autorize a respectiva NF-e. Durante este momento, o Software fará a busca das autorizações das NF-e's enviadas, informando quando a NF-e for autorizada;
* **Autorizada:** este é o estágio na qual a NF-e foi recepcionada e autorizada pela SEFAZ, podendo o contribuinte realizar a impressão do DANFE e a circulação da mercadoria. A NF-e já apresenta validade jurídica, e os dados nela contidos são de responsabilidade do emissor;
* **Cancelada:** após ser autorizada pela SEFAZ, o contribuinte poderá solicitar o cancelamento da NF-e, caso haja necessidade. Neste caso, o contribuinte deverá também justificar o pedido de cancelamento. Depois de enviada a solicitação de cancelamento, e esta sendo homologada pela SEFAZ, a NF-e mudará a situação para **Cancelada**, não sendo possível mais a sua utilização ou a impressão do respectivo DANFE;
* **Denegada:** condição que ocorre quando a NF-e é enviada para a SEFAZ e durante o processamento da nota no sistema da Fazenda verifica-se que o emitente ou destinatário está em situação de pendência perante o Fisco. Neste caso, não será permitida a impressão do DANFE e nem tão pouco o seu cancelamento;
* **Rejeitada:** após o envio e processamento da NF-e na SEFAZ, o arquivo poderá ser rejeitado caso haja alguma pendência ou problema com a NF-e. Neste caso, o sistema da SEFAZ não autorizará a NF-e, disponibilizando o motivo da rejeição. Após verificar a pendência ocorrida, o usuário poderá novamente editar a NF-e (a voltará para a situação **Em Digitação**) para realizar a correção e, após a validação e a assinatura, retransmitir para a SEFAZ;
* **Pendente de retorno da SEFAZ:** ocorre quando houve algum problema com a transmissão da NF-e para a SEFAZ (ou seja, não é conhecido se a NF-e foi enviada com sucesso para a SEFAZ) ou quando houve algum problema com a busca pelo retorno de autorizações da SEFAZ. Nestes casos, o Software irá realizar a busca das autorizações destas NF-e's na SEFAZ e caso a NF-e esteja autorizada na SEFAZ, o Software irá automaticamente mudar a situação da mesma para Autorizada.

#### Tipo de evento <a href="#tipo-de-evento" id="tipo-de-evento"></a>

O evento é qualquer fato relacionado com uma NF-e, normalmente ocorrido após a autorização de uso. A carta de correção eletrônica e o cancelamento são exemplos de eventos da NF-e e serão explicados a seguir.

* **Cancelamento:** o cancelamento da NF-e é realizado apenas em NF-e's autorizadas e serve para informar a SEFAZ que aquela NFe não tem mais valor fiscal. No sistema emissor de notas só é possível realizar o cancelamento até 24 horas da autorização da mesma. Após as 24 horas, deve ser solicitado o cancelamento, através do cancelamento extemporâneo e logo depois, o cancelamento é realizado no sistema emissor de notas;
* **Carta de correção eletrônica:** após a autorização da NF-e, é possível corrigir algumas informações através da carta de correção eletrônica. A correção de informações só é possível se o erro não estiver relacionado com: **1** - as variáveis que determinam o valor do imposto tais como: base de cálculo, alíquota, diferença de preço, quantidade, valor da operação (para estes casos deverá ser utilizada NF-e Complementar); **2** - a correção de dados cadastrais que implique mudança do remetente ou do destinatário; **3** - a data de emissão da NF-e ou a data de saída da mercadoria.

#### Eventos da manifestação do destinatário <a href="#eventos-da-manifestacao-do-destinatario" id="eventos-da-manifestacao-do-destinatario"></a>

Após a emissão da NF-e, o destinatário pode ter algumas ações sobre ela, que são eventos de manifestação. Estes serão explicados a seguir:

* **Confirmação da operação:** o evento de confirmação da operação pelo destinatário confirma a operação e o recebimento da mercadoria (para as operações com circulação de mercadoria). Se ocorrer a devolução total ou parcial das mercadorias, além do procedimento atual de geração da nota fiscal de devolução, também poderá ser comandado o evento da confirmação da operação. O registro deste evento libera a possibilidade da empresa efetuar o download da NF-e;&#x20;
* <mark style="color:red;">**Observação:**</mark> após a confirmação da operação pelo destinatário, a empresa emitente fica automaticamente impedida de cancelar a NF-e.
* **Desconhecimento da operação:** uma empresa pode ficar sabendo das operações destinadas a um determinado CNPJ consultando o serviço de consulta da relação de documentos destinados ao seu CNPJ. O evento de desconhecimento da operação permite ao destinatário informar o seu desconhecimento de uma determinada operação que conste nesta relação, por exemplo;
* **Operação não realizada:** em algumas situações, a empresa destinatária informa que a operação não foi realizada (com recusa de recebimento da mercadoria e outros motivos), não cabendo neste caso a emissão de uma nota fiscal de devolução. Este evento permite o registro da declaração de operação não realizada pelo destinatário, permitindo também a informação complementar da justificativa desta informação;
* **Ciência da emissão:** neste evento, o destinatário declara ter ciência sobre uma determinada operação destinada ao seu CNPJ, mas não possui elementos suficientes para apresentar a sua manifestação conclusiva sobre a operação citada. O registro deste evento libera também a possibilidade da empresa efetuar o download da NFe. O evento de ciência da emissão é um evento opcional e pode ser evitado, já que normalmente o destinatário da NF-e deve possuir o arquivo XML da NF-e enviado e/ou disponibilizado pelo emitente. Após um período determinado, todas as operações com ciência da emissão deverão obrigatoriamente ter a manifestação final do destinatário declarada em um dos eventos de confirmação da operação, desconhecimento ou operação não realizada.

### Inutilização <a href="#inutilizacao" id="inutilizacao"></a>

A inutilização de número de NF-e tem a finalidade de permitir que o emissor comunique à SEFAZ, até o décimo dia do mês subsequente, os números de NF-e que não serão utilizados em razão de ter ocorrido uma quebra de sequência da numeração da NF-e. A inutilização de número só é possível caso a numeração ainda não tenha sido utilizada em nenhuma NF-e (autorizada, cancelada ou denegada).

### Modalidades - Porque? <a href="#modalidades" id="modalidades"></a>

No sistema Dmaster, existem algumas modalidades em que é possível realizar a emissão de uma NF-e. Estas serão citadas e explicadas a seguir.

* **Pedidos:** essa modalidade é utilizada quando é realizada uma venda de pedido no sistema Dmaster Farma/Shop. Esse tipo de venda não é gerado cupom fiscal, apenas cria o registro no módulo emissor de nota fiscal eletrônica. Nessa modalidade, o destinatário é o cliente, que deve ser previamente cadastrado no sistema Dmaster Farma/Shop e identificado no momento da venda;
* **Convênios:** essa modalidade é utilizada ao realizar o fechamento de um convênio empresa. Após o fechamento é criado um registro no módulo emissor de nota fiscal eletrônica. Nessa modalidade, o destinatário é o convênio, que é cadastrado previamente no sistema Dmaster Farma/Shop;
* **Vendas Cupom:** essa modalidade é utilizada para qualquer venda onde é gerado cupom fiscal. Após a finalização da venda, é criado um registro no módulo emissor de nota fiscal eletrônica. Nessa modalidade, o destinatário é o cliente que deve ser previamente cadastrado no sistema Dmaster Farma/Shop. Nesse caso, o cliente pode ser identificado tanto no momento da venda no Dmaster Farma/Shop, quanto no momento da emissão da nota fiscal eletrônica;
* **Devol. Fornecedor:** essa modalidade é utilizada quando é realizado o processo de devolução para fornecedor. Ao realizar a devolução de item, é criado um registro no módulo emissor de nota fiscal eletrônica. Nessa modalidade, o destinatário é o fornecedor que deve ser previamente cadastrado no sistema Dmaster Farma/Shop;
* **Devol. Venda:** essa modalidade é utilizada quando é realizada a devolução de uma venda, onde o prazo para cancelamento do cupom fiscal foi excedido. Ao realizar a devolução da venda, é criado um registro no módulo emissor de nota fiscal eletrônica; Nessa modalidade, o destinatário é o próprio estabelecimento, não sendo necessário nenhum cadastro prévio no sistema Dmaster Farma/Shop;
* **Outras Saídas:** essa modalidade é utilizada quando é realizada baixa de estoque no sistema Dmaster Farma/Shop, por exemplo, baixa por vencimento. Após o processo, é criado um registro no módulo emissor de nota fiscal eletrônica; Nessa modalidade, caso não seja informado nenhum destinatário através do menu [Dados complementares](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/documentacao-tecnica/farma-shop/fiscal#dados-complementares), o destinatário da NF-e será o próprio estabelecimento, não sendo necessário nenhum cadastro prévio no sistema Dmaster Farma/Shop;
* **Transferências:** essa modalidade é utilizada quando é realizada uma transferência entre lojas do tipo **Por Nota Fiscal** no sistema Dmaster Farma/Shop. Após o processo, é criado um registro no módulo emissor de nota fiscal eletrônica. Nessa modalidade, o destinatário é a loja de destino, que deve ser cadastrada previamente no sistema Dmaster Farma/Shop através do cadastro de clientes.

### Finalidades - Praque? <a href="#finalidades" id="finalidades"></a>

As finalidades são utilizadas para definir qual o tipo de operação e informações fiscais serão inseridas na nota fiscal eletrônica. Algumas das finalidades padrões serão citadas e explicadas a seguir.

* **Substituição de cupom fiscal:** essa finalidade é utilizada para gerar notas fiscais da modalidade convênios e vendas cupom, ou seja, vendas onde foram gerados cupons fiscais de consumidor final;
* **Venda de mercadoria dentro do estado:** essa finalidade é utilizada para gerar notas fiscais da modalidade pedidos, onde o destinatário é do mesmo estado do emitente;
* **Venda de mercadoria fora do estado:** essa finalidade é utilizada para gerar notas fiscais da modalidade pedidos, onde o destinatário é de estado diferente do emitente;
* **Devolução de compra dentro do estado:** essa finalidade é utilizada para gerar notas fiscais da modalidade devolução fornecedor, onde o destinatário é do mesmo Estado do emitente;
* **Devolução de compra fora do estado:** essa finalidade é utilizada para gerar notas fiscais da modalidade devolução fornecedor, onde o destinatário é de Estado diferente do emitente;
* **Transferência entre filiais:** essa finalidade é utilizada para gerar notas fiscais da modalidade transferências;
* **Devolução de venda:** essa finalidade é utilizada para gerar notas fiscais da modalidade devolução de venda;
* **Perda de produto:** essa finalidade é utilizada para gerar notas fiscais da modalidade outras saídas.

#### Utilização - Primeiro envio <a href="#utilizacao-primeiro-envio" id="utilizacao-primeiro-envio"></a>

* Apenas produtos que possuem código de barras no cadastro ou código iMendes vinculado serão enviados;
* O código de barras do produto deve ser o principal e não o vinculado.

Para o primeiro envio de produtos, é utilizado o tipo de operação **Atualizar cadastro de produtos completo**. ### Exportação de produtos <a href="#exportacao-de-produtos" id="exportacao-de-produtos"></a>

{% tabs %}
{% tab title="Exportação de produtos" %}
O módulo de exportação de produtos poderá ser utilizado em vários cenários, para que sejam exportados para uma planilha as informações dos produtos cadastrados no sistema, e caso seja necessário, o cliente pode ajustar informações e posteriormente, importar a planilha ajustada para o sistema DMASTER.

**Observações:**

* Caso seja necessário realizar ajustes através da planilha gerada para importar no sistema DMASTER, não é permitido alterar a estrutura da planilha, pois assim, ela pode se tornar inválida para ser importada.
* Dentro da planilha é possível excluir linhas e ordenar as linhas por ordem alfabética dos produtos, mas conforme foi citado anteriormente não é permitido alterar a estrutura do arquivo, sendo assim, a ordem das colunas não pode ser alteradas e muito menos excluídas.

Antes de realizar a exportação da planilha, é preciso selecionar o tipo de exportação:

* **Cadastro de produtos:** exportar todos os produtos cadastrados, válidos e inválidos;
* **Cadastro de produto inválidos:** exportar apenas os produtos inválidos;
* **Consulta geral (Ativos e inativos):** exportar os produtos ativos e inativos.
