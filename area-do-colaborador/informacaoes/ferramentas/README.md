# Ferramentas

### GitBook <a href="#gitbook" id="gitbook"></a>

O GitBook é a ferramenta utilizada pela DMASTER para armazenar todas as informações que podem auxiliar o colaborador durante sua jornada, ou seja, é a plataforma de **base de dados de conhecimento.**

O acesso é através do link: [https://dmaster.gitbook.io/](https://dmaster.gitbook.io/).

### Formulário de sugestões <a href="#formulario-de-sugestoes" id="formulario-de-sugestoes"></a>

Ferramenta utilizada para que os consultores da equipe registrem sugestões referente aos sistemas DMASTER e/ou processos internos. As sugestões enviadas serão analisadas pelo gestor e se for viável, serão acatadas. É necessário que, antes de enviar uma sugestão, o consultor tenha noção sobre os sistemas/processos e suas limitações para que não sejam criadas demandas desnecessárias.

O acesso ao formulário é através do link: [https://forms.office.com/r/AALU7pqLDC](https://forms.office.com/r/AALU7pqLDC)

### Domínio interno <a href="#dominio-interno" id="dominio-interno"></a>

A DMASTER possui um domínio interno onde cada colaborador é inserido. Ao ser inserido no domínio, para acessar seu computador de trabalho, será necessário inserir usuário e senha. O usuário, por padrão é **"nome.sobrenome"** e será cadastrado previamente pelo gestor e a senha será criada pelo próprio colaborador no primeiro acesso. Através do domínio é possível ter acesso ao Servidor Arquivos, que será explicado abaixo.

#### Servidor Arquivos <a href="#servidor-arquivos" id="servidor-arquivos"></a>

A DMASTER possui um servidor, onde ficam armazenados todos os arquivos/ferramentas necessários para o dia a dia de atendimentos. Esse servidor é acessado pela rede digitando **\\\arquivos** no menu executar. Ao acessá-lo é possível visualizar várias pastas, porém as mais utilizadas pela equipe de suporte são: **"Suporte"** e **"Temporário"**. A **pasta Suporte** é onde ficam armazenadas todas as ferramentas e arquivos necessários para os atendimentos. A **pasta Temporário** é utilizada para armazenar arquivos que são temporários, ou seja, **depois de 5 dias** naquela pasta, o **arquivo é apagado**. Geralmente é utilizada pra guardar arquivos que serão utilizados posteriormente em instalações e/ou análises de erros dos sistemas.

### Controle <a href="#controle" id="controle"></a>

Sistema de gerenciamento da DMASTER, chamamos internamente de Controle. É a ferramenta utilizada por todos os colaboradores para visualizar as informações dos atendimentos. No controle é possível visualizar as informações cadastrais e os chamados de cada cliente, realizar agendamentos de chamados, registrar ponto de pausas, início e fim de jornada. Para iniciar o controle deve ser registrado ponto de início de jornada e antes de fechá-lo deve ser registrado ponto de fim de jornada. As pausas também devem ser registradas, de acordo com a seção [Pausas](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/informacoes/jornada-de-trabalho#pausas).

### Team Viewer <a href="#team-viewer" id="team-viewer"></a>

Ferramenta utilizada para possibilitar acessar remotamente o computador do cliente que está em atendimento.

A DMASTER possui uma senha padrão para acessar o computador dos clientes pelo team viewer, para que seja mais fácil a conexão, **mas o cliente não tem essa informação**, é necessário que o cliente informe a senha gerada pelo seu Team Viewer, mesmo que consiga acesso com a senha padrão. A senha padrão utilizada é **Dma21021** e deve ser configurada no computador de cada cliente no primeiro acesso.

### EzPoint Mobile <a href="#ezpoint-mobile" id="ezpoint-mobile"></a>

Aplicativo para celular que pode ser utilizado para visualizar e ajustar os registros de ponto do colaborador.

Os dados para acesso ao aplicativo são disponibilizados no momento do onboarding do colaborador a equipe DMASTER.

### Solicitação de ajuda <a href="#solicitacao-de-ajuda" id="solicitacao-de-ajuda"></a>

Ferramenta utilizada para solicitar ajuda a outros colaboradores ou do suporte avançado quando a solicitação do cliente não pôde ser resolvida pelo colaborador que está realizando o atendimento. A ferramenta foi criada para que o colaborador não fique preso naquele atendimento e para que não disperse outra pessoa da equipe com sua dúvida, se ele estiver realizando outra função. Para criar uma solicitação de ajuda é utilizado um formulário que deverá ser preenchido com atenção, com as devidas informações necessárias.

Orientações de como registrar uma [Solicitando ajuda](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/orientacoes-para-atendimentos/procedimentos-no-chamado/entregando-a-solucao/solicitando-ajuda).

### Microsoft 365 <a href="#microsoft-365" id="microsoft-365"></a>

Nós da DMASTER utilizamos os serviços da Microsoft para realizarmos toda a gestão de e-mail, arquivos e comunicação interna, sendo assim, os principais serviços utilizados são:

#### Teams <a href="#teams" id="teams"></a>

Serviço de comunicação utilizado para que a equipe possa se comunicar durante o período de jornada de trabalho. O Teams possui vários grupos. Cada um deles trata de um assunto/setor. O objetivo dessa ferramenta é facilitar a comunicação entre todos os colaboradores da DMASTER e permitir o compartilhamento de informações/dúvidas úteis e importantes.

#### Outlook <a href="#outlook" id="outlook"></a>

Serviço de gestão de e-mail utilizado para gerenciar o e-mail corporativo do Suporte e o e-mail corporativo individual de cada colaborador. O e-mail corporativo individual será criado no início do contrato de trabalho e através dele serão recebidas informações e documentos, como citado na seção [Assinatura eletrônica](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/informacoes/rotinas#assinatura-eletronica). Para realizar login, será utilizado o e-mail: **nome.sobrenome@dmaster.com.br** e a senha será definida pelo próprio colaborador. Para mais informações sobre o e-mail corporativo do Suporte, acessar a seção [E-mail Suporte](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/processos-internos/e-mail-suporte).

#### One Drive <a href="#one-drive" id="one-drive"></a>

Serviço de armazenamento de arquivos que deve ser utilizado para salvar arquivos que o colaborador julga necessário, uma vez que, **não é recomendado salvar arquivos no seu computador de trabalho**, porque periodicamente eles passam por manutenção e formatação, podendo ocorrer perda de arquivos. O One Drive é utilizado juntamente como e-mail corporativo.

### Utilização do telefone <a href="#utilizacao-do-telefone" id="utilizacao-do-telefone"></a>

A DMASTER possui um headset para realizar as ligações para os clientes e para utilizá-lo possui algumas especificidades, que serão detalhadas a seguir:

Para realizar ligações externas, é preciso primeiro discar 0, para que o telefone dê linha e logo após discar o número de telefone desejado. **Por exemplo:** **0** 30556261.

Para realizar ligações **para outro DDD**, ou para cidades que necessitam do mesmo, é preciso discar 0 + 012 + DDD(apenas com dois dígitos) + número do telefone. **Por exemplo: 0 012** 32 32814965.

Caso algum ramal esteja recebendo ligação e seja necessário atendê-la sem ir até o telefone que está tocando, é possível puxar aquela ligação através do comando **16 + número do ramal. Por exemplo:** **16** 2465.

Para saber o número do seu ramal, é utilizado o comando **#100.**

Nas estações de trabalho, é instalado um aplicativo chamado **contaction**, onde é possível visualizar o número de todos os ramais existentes e a quem ele pertence. Caso seja necessário ligar para algum ramal, é necessário apenas digitar o número do ramal desejado.

Anteriormente, as solicitações de atendimento dos clientes eram através de ligação ou abertura de chamado via site ou sistema. Existia uma URA que direcionava a ligação para o setor correto, de acordo com opção selecionada. Hoje, as solicitações de atendimento são exclusivamente via chamado, e nós entramos em contato com o cliente. Sendo assim, o cliente abre um chamado via site ou sistema e nós entramos em contato através do telefone.

#### Criar sala de conferência <a href="#criar-sala-de-conferencia" id="criar-sala-de-conferencia"></a>

A sala de conferência é criada com o intuito de inserir mais de duas pessoas em uma só ligação. Dessa forma, será possível dois consultores participarem de uma mesma ligação com o cliente. Abaixo será descrito o processo para ser criada a sala de conferência.

1. Ligar para o autor 1;
2. Após o autor 1 atender a ligação, utilizar a tecla **Flash**, dessa forma, o autor 1 ficará com a chamada em espera;
3. Ligar para o autor 2;
4. Após o autor 2 atender a ligação, utilizar a tecla **Flash** e logo após utilizar **Flash + 19** para que a a ligação seja unificada e assim, os consultores estarão na mesma ligação, porém cada um com o seu telefone.
